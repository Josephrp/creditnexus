## FDC3 2.0 Compliance

CreditNexus is fully compliant with the **FDC3 2.0** (Financial Desktop Connectivity and Collaboration Consortium) standard, enabling seamless interoperability with other financial desktop applications.

**Code Reference**: `client/src/context/FDC3Context.tsx` (lines 234-492), `client/src/hooks/useFDC3.ts`, `openfin/fdc3-intents.json`

## FDC3 Implementation

### Context Broadcasting

CreditNexus broadcasts FDC3 contexts for seamless data sharing:

```typescript
// Broadcast credit agreement context
await broadcast({
  type: 'fdc3.creditnexus.agreement',
  agreement: creditAgreementData
});
```

**Code Reference**: `client/src/context/FDC3Context.tsx` (lines 320-333)

### Intent Handling

CreditNexus listens for and raises FDC3 intents:

**Supported Intents**:
- `ViewLoan` - View loan details
- `ViewDeal` - View deal information
- `ExtractDocument` - Extract document data
- `ExecuteTrade` - Execute trade

**Code Reference**: `openfin/fdc3-intents.json`, `client/src/context/FDC3Context.tsx` (lines 339-353)

### App Channels

CreditNexus uses custom app channels for workflow orchestration:

- `creditnexus.workflow` - Workflow events
- `creditnexus.extraction` - Document extraction events
- `creditnexus.portfolio` - Portfolio updates

**Code Reference**: `client/src/context/FDC3Context.tsx` (lines 258-274)

## Context Types

CreditNexus defines custom context types in the `finos.creditnexus.*` namespace:

### Credit Agreement Context

```typescript
{
  type: 'fdc3.creditnexus.agreement',
  agreement: {
    dealId: string,
    borrower: string,
    amount: number,
    // ... CDM-compliant data
  }
}
```

### Loan Context

```typescript
{
  type: 'fdc3.creditnexus.loan',
  loan: {
    loanId: string,
    status: string,
    // ... loan data
  }
}
```

## System Interoperability Flow

The platform components work as a "Chain of Command" using FDC3:

1. **Extract**: Document Parser broadcasts loan data
2. **Trade**: Trade Blotter receives signal and pre-fills trade ticket
3. **Analyze**: GreenLens picks up signal for ESG analysis
4. **Verify**: Verification Demo broadcasts breach alerts
5. **Surveil**: Risk War Room listens for alerts and highlights assets

**Code Reference**: `README.md` (lines 507-517)
